My Projects!

Pebl

The most expansive project I have worked on, Pebl. It is a basic social media client-server pairing similar to twitter, but designed with expandibility in mind. As a group collaboration we created version 1, since then I have revised the program code for some minor updates with more being planned with the original team. Through this project I was responsible for the JavaFX interface originally, although with that completed for now I have been working on revising all of the server connection code in the client and bugfixes.

ABnegation Company Network

A project I suprisingly really enjoyed working on was this network simulation. The scenario for this project was a the ABnegation Company was upgrading their network and I was tasked with building a test simulation before they committed to the expense. This involved the creation of subnets and the use of some complex routing and masks to keep the nextworks as small as possible while leaving space for expansion, including some complexities from routing DHCP packets between subnets for the external building without escaping into the wider network and configuration of manual IP's and wireless Access for other parts of the building. This is the project that has really got me interested in Network Engineering.

Scribble

Scribble started as a Java console based recreation of Scrabble, Originally with a lesser ruleset but I eventually expanded this further to the full ruleset, Including scoring for overlapping words and special board tiles. A big part of my personal additions to the game was well spent on the dynamic board sizing and other touches. As much as at it's core it's simple sounding concept, it proved to be a challenge to implement and I enjoyed making it and enjoy the finished product. I learned a lot about making intuitive command line interfaces for progams from this project too. While not as accessible as a GUI, I tried to make it as polished and easy to use as possible. The final result is I belive a very well polished recreation.

From there, I worked on modding scratch games, my dad helped me with a facebook clone with html and angular.js and I also was playing with Embedded Systems like Raspberri Pi's with python.

Time Went on and Scratch Turned into Gamemaker which became Visual Basic, I began working with Vintage PC's to understand PC hardware and teh fundementals of software. By highschool I was intimately familiar with PC's and electronics, punching up doing work well above my level to a high standard.

Now that I've built up this fundemental foundation in software and hardware, I'm putting it to use in my degree learning Java and C/C++, working with Pi Pico's in micropython and in my spare time working on electronics repair and building guitar pedals and amplifiers.
